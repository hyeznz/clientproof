# 잘 안 됐던 함정들 (Pitfalls)

같은 실수를 두 번 반복하지 않기 위한 기록. 각 항목마다 **증상 → 원인 → 해결**.

---

## 1. JS `offsetWidth`의 함정 (block 요소)

**증상**: JS가 텍스트 폭을 측정해도 카드에서 글자가 잘림  
**원인**: 
- `.kw-line`이 `display: block`이면 부모 폭을 꽉 채움
- 텍스트가 짧아도 `offsetWidth` = 부모 폭 반환
- JS가 "이미 맞음"이라고 판단 → FS 축소 안 함  
**해결**:
- `.cell-keyword`를 `display: flex; flex-direction: column; align-items: flex-start` 
- → `.kw-line`(flex child)이 콘텐츠 폭만큼만 차지
- `Math.max(offsetWidth, scrollWidth)` 이중 안전장치

---

## 2. `max-width: 100%`가 측정 왜곡

**증상**: nowrap 텍스트가 카드 오른쪽으로 넘쳐도 JS가 못 감지  
**원인**:
- `max-width: 100%` + `white-space: nowrap` = 박스는 부모 폭 캡, 콘텐츠만 오버플로
- `offsetWidth`가 캡된 박스 크기 반환  
**해결**:
- `.kw-line`에서 `max-width` 제거
- 박스가 자연 콘텐츠 폭으로 확장 → 정확한 측정

---

## 3. `letter-spacing` 단위 (em vs px)

**증상**: 폰트 크기 바꿨는데 자간 비율이 안 맞음  
**원인**: 
- `letter-spacing: px`는 절대값 → 폰트 사이즈 바뀌어도 불변
- `letter-spacing: em`은 폰트 크기에 비례  
**해결**:
- 메인 타이틀: `em` 단위 (`-0.03em`)로 비율 유지
- blockFit의 폰트 변경 시 자동 스케일 됨

---

## 4. `line-height: 0.96`으로 인한 상단 클리핑

**증상**: 첫 줄의 어센더(위쪽 삐침)가 잘림  
**원인**:
- line-height < 1.0 → 라인 박스가 폰트보다 짧음
- 어센더가 박스 위로 튀어나감
- 부모에 `overflow: hidden`이 걸려있으면 잘림  
**해결**:
- line-height 최소 1.0 이상 (권장 1.1~1.2)
- 메인 타이틀은 1.15, 카드 제목은 1.2

---

## 5. `getComputedStyle('--pad')`가 `clamp()` 값을 resolve 안 함

**증상**: JS가 `--pad`를 읽어도 `NaN` 반환  
**원인**:
- CSS 커스텀 프로퍼티가 `clamp(9px, 4.8vw, 18px)` 같은 식이면
- `getPropertyValue('--pad')`는 원본 문자열 반환 ("clamp(...)")
- `parseFloat()`가 "clamp" → NaN  
**해결**:
- 샘플 카드의 `paddingLeft`를 실측으로 읽기
- `parseFloat(getComputedStyle(sampleCard).paddingLeft)` → 실제 resolve된 px 값

---

## 6. 탭 전환 시 FS 달라지는 문제

**증상**: 책임감 탭만 유독 글자 크게/작게 나옴  
**원인**:
- JS fitCardTitles가 "현재 렌더된 탭의 카드만" 측정
- 탭마다 카드 조합 다름 → minFS 다름  
**해결**:
- 전역 계산: 모든 탭의 arrangement 순회해 글로벌 minFS 산출
- 캐시(`_globalCardFS`)에 저장, 탭 전환 시 재사용

---

## 7. Word gap 과다 (letter-spacing 중첩)

**증상**: "흔치 | 않아요" 사이 간격이 너무 벌어짐  
**원인**:
- JS가 letter-spacing 추가 시 공백 char 앞뒤로도 자간이 붙음
- 실제 단어 사이 간격 = 자연 공백 + 양쪽 letter-spacing (2배 효과)  
**해결**:
- `.mt-line--tight { word-spacing: -0.1em }`로 공백에 음수 보정
- 단어 사이 간격을 자연스럽게 좁힘

---

## 8. 짧은 줄이 왼쪽으로 튀어나와 보임

**증상**: "흔치 않아요"가 상단 줄 대비 왼쪽에 더 나와 보임  
**원인**:
- 상단 줄은 따옴표(`'`)로 시작 → 글리프에 left side-bearing 존재
- 짧은 줄은 한글로 시작 → side-bearing 없음
- 시각적으로 짧은 줄이 더 왼쪽에 있는 느낌  
**해결**:
- `.mt-line--tight { padding-left: 0.04em }` — 폰트 크기 비례 미세 들여쓰기

---

## 9. CSS Setext H2 문법 (마크다운)

**증상**: 옵시디언에서 후기 마지막 문장이 헤더처럼 크게 보임  
**원인**:
- 마크다운: 텍스트 바로 아래 `---`는 Setext H2 문법
- 그 텍스트가 `<h2>`로 변환되어 폰트 큼  
**해결**:
- `---` 앞에 항상 빈 줄 1개 보장
- 빈 줄 있으면 `---`는 horizontal rule로 해석

---

## 10. 조사 부자연스러움 (GS → 이사짐캐리)

**증상**: "이사짐캐리을", "이사짐캐리이었어요", "이사짐캐리이" 등 어색한 조사  
**원인**:
- 원본 "GS"는 받침(자음 종성)으로 취급되어 "을/이/이었/은" 조사 사용
- "이사짐캐리"는 '리'(모음 종성)로 끝나 "를/가/였/는" 조사가 자연스러움
- 단순 "GS" → "이사짐캐리" 치환 시 조사가 받침용 그대로 남음  
**해결**:
- 별도 패스로 조사 교정 (`scripts/fix_particles.py`)
- 패턴: `이사짐캐리을` → `이사짐캐리를`, `이사짐캐리이었` → `이사짐캐리였` 등

---

## 11. UTF-8 바이트 잘림

**증상**: 파일 끝에 ♡(3바이트) 중 2바이트만 쓰여 파일 파손  
**원인**:
- Python에서 write 시 불완전한 바이트 시퀀스 남는 버그 재현
- 이유 불명, 특정 인코딩 경계 케이스  
**해결**:
- 백업에서 복원
- `data.encode('utf-8')` → `write(bytes)` 명시적 바이너리 쓰기로 검증

---

## 12. 벤토 패턴 "십자 간격" 착각

**증상**: 큰 카드에 짧은 제목이 들어가 상단에 빈 공간 → 패턴 깨져 보임  
**원인**:
- 좌측 하단 정렬(공식 ③) 때문에 짧은 텍스트는 카드 상단에 빈 공간 남김
- 인접 카드와 높이 달라 L자/십자 간격처럼 보임  
**해결**:
- 실제로는 패턴 깨짐이 아님 (디자인 의도된 결과)
- DESC zip 재배정으로 큰 카드에 긴 제목이 자주 배치되면 덜 도드라짐

---

## 13. 하이라이트 탭 태그 중복 카운트

**증상**: id:73 (실력의차이+하이라이트)처럼 하이라이트 탭 겸 메인 탭에 있는 후기  
**원인**:
- 단순 "tags.length ≥ 2"로 중복 판정하면 하이라이트와의 공존도 중복으로 계산  
**해결**:
- `isDuplicateAcrossTabs(r)` = "하이라이트 제외" 태그 수 ≥ 2로 판정
- 하이라이트는 "쇼케이스" 탭이라 중복 카운트에서 제외

---

## 14. 카드 폭 JS 계산과 실제 렌더 불일치

**증상**: JS 계산 availW와 실제 카드 clientWidth가 몇 픽셀 다름  
**원인**:
- `(viewport - 2*edge - gap) / 2` 공식은 body max-width 고려 안 함
- 데스크탑은 body가 720px 캡 → 수식대로면 overshoot  
**해결**:
- 샘플 카드의 `clientWidth`를 실측 우선 사용
- CSS-derived는 폴백 (초기 렌더 전 호출 시)

---

## 15. 문단 매칭의 시그니처 위치

**증상**: 마크다운 문단 재구성 시 라인 중간에 잘못 break 삽입됨  
**원인**:
- fullTextMap 문단 시작 시그니처가 마크다운 라인 중간에 등장
- "침착하게 잘 해결을 해주셨고 저희집에서 어머니..." — "저희집에서"가 P3 시작
- 이 라인 앞에 break 넣으면 "침착하게" 문장이 뚝 끊어짐  
**해결**:
- 시그니처 위치가 라인 시작 8자 이내일 때만 브레이크 인정
- 중간에 있으면 스킵 (해당 문단 경계 포기)
