# CLAUDE.md

이 파일은 Claude가 이 프로젝트에서 작업할 때 **가장 먼저 읽고 준수해야 하는 지침**입니다.

---

## 🎯 프로젝트 개요

- **프로젝트명**: ClientProof (고객 후기 벤토 그리드 웹페이지)
- **메인 산출물**: `reviews_v{메이저}w{마이너}.html` (예: `reviews_v04w27.html`)
- **현재 최신 버전**: `reviews_v04w27.html` ← 수정 작업 시 이 파일을 기준으로 새 버전 생성
- **문서 폴더**: `docs/` — 작업 가이드 및 설계 문서 보관

---

## 📂 폴더 구조 규칙

```
ClientProof/
├── CLAUDE.md                  ← 이 파일 (절대 이동/삭제 금지)
├── index.html                 ← 초기 랜딩 페이지 (보조)
├── reviews_v04wNN.html        ← 최신 작업 버전 (루트에 1개만 유지)
├── reviews_v04w(NN-1).html    ← 바로 전 버전 (빠른 롤백용, 1개만 유지)

├── docs/                      ← 가이드 문서
├── font/, font_original_pfb/  ← 폰트 (수정 드묾)
└── resources/                 ← 후기 원본 마크다운
```

## 🔄 자동 아카이브 규칙 (매우 중요)

### 새 버전 만들 때 Claude가 자동으로 수행할 작업

사용자가 "새 버전 만들어줘", "수정해줘", "v04w28로 해줘" 등 **HTML 수정 요청**을 하면,
Claude는 **사용자가 별도로 요청하지 않아도** 다음 순서를 자동 실행:

1. **최신 버전 확인**
   ```bash
   ls reviews_v04w*.html  # 루트에서 최신 번호 파악 (예: v04w27)
   ```

2. **새 버전 파일 생성** (기존 최신본을 복사)
   ```bash
   cp reviews_v04w27.html reviews_v04w28.html
   ```

3. **루트에 2개 초과되면 이전-이전 버전을 아카이브로 이동**
   - **규칙**: 루트에는 항상 **최신 2개 버전만** 유지 (최신 + 바로 전)
   - 예시: v04w28을 만들었다면 루트엔 `v04w27`, `v04w28` 2개만 남기고
     `v04w26`은 `archive/v04_history/`로 자동 이동
   ```bash
   # 새 버전 만든 직후에 실행
   # 루트에 reviews_v04w*.html이 3개 이상이면 가장 오래된 것을 이동
   mv reviews_v04w26.html archive/v04_history/
   ```

4. **새 버전(v04w28)을 수정**
   - 기존 버전(v04w27)은 절대 직접 수정하지 않음

5. **작업 완료 후 사용자에게 보고**
   - "새 버전 `reviews_v04w28.html` 생성 완료"
   - "이전 버전 `reviews_v04w26.html`은 `archive/v04_history/`로 이동"

### ⚠️ 예외 처리

- **사용자가 "아카이브 하지 마" 또는 "임시로 비교용으로 남겨둬"라고 말하면** → 이동 보류
- **프로토타입(v01~v03)은 절대 건드리지 않음** — 이미 archive에 있고 구조가 다름
- **`index.html`은 별도 취급** — 버전 관리 대상 아님, 루트에 그대로 유지

## 📐 작업 시 원칙

1. **기존 HTML 직접 수정 금지** — 항상 복사 후 새 버전에서 작업
2. **한 번의 작업 단위 = 한 버전 증가** — 한 세션에 여러 수정이 있어도 가능한 같은 버전에서 마무리
3. **docs 문서 업데이트** — 구조나 워크플로우가 바뀌면 `docs/FILE_MAP.md`, `docs/WORKFLOW.md` 갱신
4. **Git 상태 보고** — 큰 작업 마치면 "git에 커밋/푸시 할까?" 질문으로 마무리
